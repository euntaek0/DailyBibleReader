# Functional Requirements

## FR-1. 음성 입력

- 사용자는 마이크를 통해 한국어 음성을 입력할 수 있어야 한다.
- 브라우저 내에서 음성 입력이 이루어져야 한다.

## FR-2. 음성 → 텍스트 변환

- Web Speech API를 사용하여 음성을 실시간으로 텍스트로 변환한다.
- 언어 설정은 ko-KR로 고정한다.

## FR-3. Verse 읽기 여부 판정

- 사전에 정의된 성경 verse 텍스트와
  인식된 음성 텍스트를 비교한다.
- 단어(어절) 단위로 비교하여
  일정 비율 이상 일치 시 "읽음"으로 판정한다.

## FR-4. 판정 결과 제공

- 판정 결과는 다음 중 하나여야 한다.
  - 읽음
  - 부분 읽음
  - 읽지 않음

## FR-5. 서버 미사용

- 모든 로직은 프론트엔드에서 수행되어야 한다.
- 서버로 음성 또는 텍스트 데이터를 전송하지 않는다.

## FR-6. Mobile Responsive UI

- UI는 **Mobile First**로 설계되어야 한다.
- 모든 인터랙션 요소는 터치 친화적이어야 한다 (최소 터치 영역 확보).
- 다양한 화면 크기(Mobile, Tablet, Desktop)에 적절하게 반응형으로 동작해야 한다.
